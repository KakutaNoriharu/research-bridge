**プロジェクト名:** ResearchBridge — 産学連携マッチングプラットフォーム
**作成日:** 2026年3月31日
**バージョン:** 1.0

---

## 1. プロジェクト概要

### 1.1 背景と目的

大学・大学院と企業の共同研究は、現状以下の構造的な課題を抱えている。

- 研究者の「今できること・したいこと」がリアルタイムに企業側へ伝わらない
- キーワードベースの検索では研究の本質的な親和性が判断できない
- マッチング後の初期コミュニケーションに至るまでの障壁が高い
- 中小企業・スタートアップが大学研究者にアクセスしにくい

本サービス **ResearchBridge** は、これらの課題をAIによる意味的マッチングと軽量なコミュニケーション機能によって解決する、研究者と企業双方向の産学連携プラットフォームである。

### 1.2 ターゲットユーザー

| ユーザー種別 | 対象 |
|---|---|
| 研究者側 | 大学・大学院の研究者・教員・博士学生 |
| 企業側 | R&D部門・技術調達担当・スタートアップ創業者 |

### 1.3 差別化ポイント

既存サービス（J-OPERA、TLO等）との最大の差別化は **AIによる意味的マッチング** にある。研究概要・論文アブストラクト・技術ニーズをEmbeddingモデルでベクトル化し、キーワード一致ではなく研究内容の意味的類似度によってマッチングスコアを算出する。

## 2. システム概要

### 2.1 サービス名

**ResearchBridge**

### 2.2 システム構成（概要）

```
[フロントエンド]
  Next.js (TypeScript)

[バックエンド]
  Node.js / Express または FastAPI (Python)

[データベース]
  PostgreSQL（ユーザー・プロフィール・メッセージ）
  pgvector（Embeddingベクトルの保存・類似度検索）

[AI / Embedding]
  OpenAI text-embedding-3-small または
  Sentence-BERT（日本語対応モデル）

[認証]
  NextAuth.js / Firebase Authentication

[ホスティング]
  Vercel（フロントエンド）
  Railway / Render（バックエンド）
```

### 2.3 利用フロー概要

```
研究者登録
  → プロフィール・研究概要入力
  → Embeddingベクトル生成・保存

企業登録
  → 企業情報・技術ニーズ入力
  → Embeddingベクトル生成・保存

マッチング
  → AIスコアによるレコメンド表示
  → 検索・フィルタリング

コンタクト
  → インタレスト送信（いいね相当）
  → マッチング成立後にメッセージ解放
  → アプリ内メッセージでコミュニケーション
```

## 3. 機能要件

### 3.1 認証・アカウント管理

| ID | 機能名 | 説明 | 優先度 |
|---|---|---|---|
| AUTH-01 | ユーザー登録 | メールアドレス＋パスワードによる新規登録 | Must |
| AUTH-02 | ロール選択 | 登録時に「研究者」または「企業」を選択 | Must |
| AUTH-03 | ログイン / ログアウト | セッション管理 | Must |
| AUTH-04 | パスワードリセット | メールによるリセットリンク送信 | Must |
| AUTH-05 | OAuth連携 | Google / GitHubアカウントでのログイン | Should |

### 3.2 プロフィール機能

#### 研究者プロフィール

| ID | 項目 | 説明 | 優先度 |
|---|---|---|---|
| PRF-R-01 | 基本情報 | 氏名・所属大学・研究室・役職 | Must |
| PRF-R-02 | 研究概要 | 自由記述（最大1000文字）→ Embedding生成に使用 | Must |
| PRF-R-03 | 研究キーワード | タグ形式で最大10件 | Must |
| PRF-R-04 | 技術スタック | 保有スキル・ツール（例：C++, ns-3, MATLAB） | Must |
| PRF-R-05 | 論文・実績リンク | DOI / URL形式で最大5件 | Should |
| PRF-R-06 | 連携可能な形態 | 共同研究 / 受託研究 / 技術相談 などを選択 | Must |
| PRF-R-07 | 公開設定 | プロフィールの公開 / 非公開切り替え | Must |

#### 企業プロフィール

| ID | 項目 | 説明 | 優先度 |
|---|---|---|---|
| PRF-C-01 | 基本情報 | 企業名・業種・従業員数・担当者名 | Must |
| PRF-C-02 | 技術ニーズ | 解決したい課題・求める技術の自由記述（最大1000文字）→ Embedding生成に使用 | Must |
| PRF-C-03 | 求める研究分野 | タグ形式で最大10件 | Must |
| PRF-C-04 | 連携形態の希望 | 共同研究 / 受託 / PoC など | Must |
| PRF-C-05 | 予算規模（任意） | 年間予算感（レンジ選択） | Could |

### 3.3 AIマッチング機能

| ID | 機能名 | 説明 | 優先度 |
|---|---|---|---|
| MATCH-01 | Embedding生成 | プロフィール保存時に研究概要・技術ニーズをベクトル化し保存 | Must |
| MATCH-02 | マッチングスコア算出 | コサイン類似度によるスコア算出（0〜1.0） | Must |
| MATCH-03 | レコメンド一覧 | ログイン後トップにスコア上位10件を表示 | Must |
| MATCH-04 | スコア表示 | マッチングスコアをパーセンテージで表示 | Should |
| MATCH-05 | 再計算トリガー | プロフィール更新時にEmbeddingを自動再生成 | Must |

### 3.4 検索・フィルタリング機能

| ID | 機能名 | 説明 | 優先度 |
|---|---|---|---|
| SEARCH-01 | フリーワード検索 | キーワードによる全文検索 | Must |
| SEARCH-02 | 研究分野フィルタ | タグ・分野による絞り込み | Must |
| SEARCH-03 | 連携形態フィルタ | 共同研究・受託・相談などで絞り込み | Must |
| SEARCH-04 | 大学・地域フィルタ | 所属大学・都道府県で絞り込み | Should |
| SEARCH-05 | ハイブリッド検索 | AIスコア＋キーワードの複合ランキング | Could |

### 3.5 インタレスト・マッチング機能

| ID | 機能名 | 説明 | 優先度 |
|---|---|---|---|
| INT-01 | インタレスト送信 | 相手プロフィールに「興味あり」を送信 | Must |
| INT-02 | マッチング成立 | 双方がインタレストを送信したらマッチング成立 | Must |
| INT-03 | 通知 | インタレスト受信・マッチング成立時にメール通知 | Must |
| INT-04 | インタレスト管理 | 送受信リストの確認・取り消し | Should |

### 3.6 メッセージ機能

| ID | 機能名 | 説明 | 優先度 |
|---|---|---|---|
| MSG-01 | テキストメッセージ | マッチング成立後にアプリ内メッセージが解放 | Must |
| MSG-02 | メッセージ履歴 | 会話スレッド形式での表示 | Must |
| MSG-03 | 未読バッジ | 未読メッセージ数の表示 | Must |
| MSG-04 | ファイル添付 | PDF・画像ファイルの送受信（最大5MB） | Could |
| MSG-05 | メッセージ通知 | 新着メッセージのメール通知 | Should |

## 4. 非機能要件

### 4.1 パフォーマンス

| 項目 | 要件 |
|---|---|
| レスポンスタイム | 通常画面：1秒以内、マッチングスコア算出：3秒以内 |
| 同時接続 | 初期フェーズ：100ユーザー以下を想定 |
| Embedding処理 | プロフィール保存後、5秒以内にベクトル生成完了 |

### 4.2 セキュリティ

| 項目 | 要件 |
|---|---|
| 通信暗号化 | HTTPS（TLS 1.2以上）必須 |
| 認証 | JWTによるセッション管理、有効期限あり |
| パスワード管理 | bcryptによるハッシュ化 |
| 個人情報 | メールアドレス等はプロフィール公開対象外 |
| レート制限 | API呼び出しにレート制限を設定（DDoS対策） |

### 4.3 可用性・運用

| 項目 | 要件 |
|---|---|
| 稼働率 | 95%以上（ポートフォリオ用途のため商用SLAは不要） |
| バックアップ | DBの日次バックアップ |
| ログ | エラーログ・アクセスログの収集 |

### 4.4 アクセシビリティ・UX

| 項目 | 要件 |
|---|---|
| 対応デバイス | PC（優先）・スマートフォン（レスポンシブ対応） |
| 対応ブラウザ | Chrome / Firefox / Safari 最新版 |
| 言語 | 日本語（初期）、英語（将来対応） |

## 6. データモデル（概要）

### 主要エンティティ

```
users
  id, email, password_hash, role (researcher | company), created_at

researcher_profiles
  id, user_id, name, university, lab, position,
  research_summary, keywords[], collaboration_types[],
  embedding vector(1536), updated_at

company_profiles
  id, user_id, company_name, industry, contact_name,
  tech_needs, desired_fields[], collaboration_types[],
  embedding vector(1536), updated_at

interests
  id, sender_id, receiver_id, status (pending | matched | withdrawn),
  created_at

messages
  id, match_id, sender_id, body, is_read, created_at

matches
  id, researcher_id, company_id, matched_at
```

## 7. 技術スタック詳細

| レイヤー | 技術 | 選定理由 |
|---|---|---|
| フロントエンド | Next.js 14 (App Router) + TypeScript | SSR対応・型安全・Vercelとの親和性 |
| スタイリング | Tailwind CSS | 高速なUI構築・ポートフォリオでの可読性 |
| バックエンド | FastAPI (Python) | Embeddingライブラリとの親和性・非同期処理 |
| DB | PostgreSQL + pgvector | ベクトル検索をDBレイヤーで完結させる |
| Embedding | OpenAI text-embedding-3-small | 高精度・低コスト・日本語対応 |
| 認証 | NextAuth.js | Next.jsとのシームレスな統合 |
| デプロイ | Vercel + Railway | 無料枠でポートフォリオ公開が可能 |

## 8. 開発スコープと優先度

### MVP（最初にリリースする機能）

1. 認証・ロール別登録
2. プロフィール作成・編集（研究者・企業）
3. Embeddingによるマッチングスコア算出
4. AIレコメンド一覧表示
5. インタレスト送信・マッチング成立
6. アプリ内メッセージ（テキストのみ）

### Phase 2（MVP後に追加）

- 検索・フィルタリング機能の強化
- メール通知
- OAuth連携（Google / GitHub）
- プロフィール公開・非公開設定

### Phase 3（将来検討）

- ハイブリッド検索（キーワード＋AI）
- ファイル添付機能
- 英語対応
- 管理者ダッシュボード

## 9. 制約・前提条件

- 本サービスはポートフォリオ目的で開発する。商用運用は想定しない。
- OpenAI APIの利用料が発生するため、Embedding生成はプロフィール保存時のみとし、検索都度の呼び出しは行わない。
- 初期フェーズでは管理者機能（ユーザー管理・コンテンツモデレーション）は対象外とする。
- NDA・契約書の締結フローはスコープ外とし、アプリ外で別途実施することを前提とする。

## 10. 用語定義

| 用語 | 定義 |
|---|---|
| Embedding | テキストを数値ベクトルに変換したもの。意味的な類似度計算に使用する |
| コサイン類似度 | 2つのベクトル間の角度から類似度を算出する手法（-1〜1、1が完全一致）|
| マッチング | 研究者と企業双方がインタレストを送り合った状態 |
| インタレスト | 相手プロフィールへの「興味あり」アクション |
| pgvector | PostgreSQLのベクトル検索拡張機能 |
| MVP | Minimum Viable Product。最小限の機能で成立するプロダクト |
